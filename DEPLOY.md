# 외부망 배포 방법

이 앱을 다른 장소, 다른 핸드폰, 외부망에서 열려면 `localhost`가 아니라 인터넷에 공개된 서버에 올려야 합니다. 가장 간단한 방법은 Render 같은 Node 호스팅을 쓰는 것입니다.

## Render 배포 순서

1. `hr-secret-vote` 폴더를 GitHub 저장소에 올립니다.
2. Render에서 `New` -> `Blueprint`를 선택합니다.
3. GitHub 저장소를 연결합니다.
4. `render.yaml`을 인식하면 `ADMIN_PASSWORD` 값을 묻습니다.
5. 관리자 비밀번호로 사용할 값을 입력합니다.
6. 배포가 끝나면 `https://...onrender.com` 형태의 주소가 생깁니다.
7. 그 주소를 선생님들께 안내하면 외부망에서도 접속할 수 있습니다.

## 중요한 점

- `render.yaml`에는 영구 저장용 디스크가 포함되어 있습니다.
- 투표 데이터는 서버의 `/var/data/ballot-data.json`에 저장됩니다.
- 영구 디스크가 없는 무료 서버는 재시작/재배포 때 투표 데이터가 사라질 수 있으므로 실제 회의 운영에는 권장하지 않습니다.
- 관리자 비밀번호는 화면에 표시되지 않으며, Render 환경변수 `ADMIN_PASSWORD`로 설정합니다.

## 로컬 테스트와 외부 배포의 차이

- 내 컴퓨터에서만 테스트: `http://localhost:4173`
- 같은 와이파이 핸드폰 테스트: `http://컴퓨터IP:4173`
- 다른 장소/외부망 운영: `https://배포주소`

외부망 운영에는 마지막 방식이 필요합니다.
